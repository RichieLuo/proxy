# Selective Domain Proxy

只对指定域名的标签页走代理的 Chrome 浏览器扩展。

## 功能

- 配置白名单域名，只有匹配的域名才走代理
- 支持两种代理模式：**URL 匹配** 和 **标签页匹配**
- 支持 HTTP / HTTPS / SOCKS4 / SOCKS5 代理协议
- 工具栏图标显示代理状态（ON/OFF）

## 两种代理模式

### URL 匹配（默认推荐）

通过 PAC 脚本拦截每个请求，检查**请求目标域名**是否在白名单中，匹配则走代理。

- 优点：不影响非白名单标签页，无切换延迟
- 例：白名单有 `example.com`，访问 `example.com/api` 走代理，访问 `google.com` 不走代理

### 标签页匹配

当**当前活跃标签页**的地址栏域名在白名单中时，全局开启代理；切换到非白名单标签页时自动关闭。

- 优点：当前标签页的所有请求（包括第三方资源）都走代理
- 注意：代理是浏览器级别的，后台标签页也会受当前模式影响

## 安装

1. 打开 Chrome 浏览器，地址栏输入 `chrome://extensions/`
2. 右上角开启「开发者模式」
3. 点击「加载已解压的扩展程序」
4. 选择本项目所在目录
5. 扩展图标会出现在工具栏

## 使用

1. 点击工具栏扩展图标，打开配置弹窗
2. 填入代理服务器地址和端口（如 `127.0.0.1:8080`）
3. 选择代理协议
4. 在域名白名单中添加需要代理的域名（如 `example.com`，自动匹配子域名）
5. 选择代理模式
6. 点击「保存设置」

### 域名匹配规则

- 添加 `example.com` 会匹配 `example.com` 及所有子域名（`www.example.com`、`api.example.com` 等）
- 可以一次粘贴多个域名，用逗号、分号或空格分隔
- 支持粘贴完整 URL，会自动提取域名（去掉协议和路径）

## 项目结构

```
Proxy/
├── manifest.json      # Chrome Extension Manifest V3
├── background.js      # Service Worker：代理逻辑核心（PAC 脚本生成、标签页追踪）
├── popup.html         # 配置弹窗界面
├── popup.js           # 弹窗交互逻辑
└── icons/             # 图标文件（可自行替换）
```

## 权限说明

| 权限 | 用途 |
|------|------|
| `proxy` | 设置浏览器的代理配置 |
| `storage` | 保存用户设置（白名单、代理服务器信息） |
| `tabs` | 监听标签页 URL 变化，实现标签页匹配模式 |
